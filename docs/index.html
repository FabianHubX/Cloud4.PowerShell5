1. Einführung

Die folgende Dokumentation dient als Anleitung in der Benutzung des PowerShell Modules für den Zugriff auf die Public API der Cloud 4.0 Plattform https://portal.swiss.cloud/ der HIAG Data.

Die aktuelle Version des Modules sowie der Source Code ist auf GitHub abrufbar:

https://github.com/HiagData/Cloud4.PowerShell5

2. Installation des PowerShell Modules

Die Installation des PowerShell Modules geschieht durch Downloaden aller aktuell kompilierten Dateien auf welche sie unter dieser Adresse finden:

https://github.com/HiagData/Cloud4.PowerShell5/releases

Diese ZIP herunterladen, entpacken und komplett in einem Ordner ihrer Wahl ablegen.

Starten Sie eine und importieren Sie das Module:

Import-Module "<Ordner>\Cloud4.Powershell5.Module.psd1"

  1. Verbinden mit der Cloud 4.0

Die Verbindung mit der Cloud 4.0 erfolgt über das Kommando:

Open-Cl oud4Connection

 

Folgende Parameter müssen mitgegeben werden:

-Credential               Anmeldedaten in Form eines PSCredential Objekts

-LoginUrl                  "https://login.swiss.cloud"

-ApiUrl                       "https://api.swiss.cloud"

-TenantId                   TenantID als GUID

 

Die TenantID stellt die aktuelle ID des Tenants in der Cloud 4.0 dar. Zu einem späteren Zeitpunkt können weitere Tenant erstellt werden. Um die aktuelle Liste der verfügbaren Tenants zu erhalten, ist das “Open-Cloud4Connection” Kommando ohne den Parameter “TenantId” ausführen. Dies liefert die aktuelle Liste an Tenants zurück. Wird keine TenantId angegeben und es ist nur ein Tenant verfügbar wird dieser automatisch gesetzt.

 

Für die Erstellung eines PSCredential Objekts kann folgender PowerShell Code verwendet werden:

$username = "e-mail addresse"

$password = 'Password' | ConvertTo-SecureString -asPlainText -Force

$cred = New-Object System.Management.Automation.PSCredential($username,$password)

 

Bei erfolgreicher Anmeldung wird folgendes Resultat zurückgegeben:

Connection established

UserName               LogonUrl              ApiUrl             TenantId

--------                            --------                        ------                     --------

(User)                    (LoginUrl)             (ApiUrl)                 (TenantID)

 

 

 

  • Tenant verwalten

Mit folgendem Kommando kann der entsprechende Tenant gesetzt werden:

Set-Cloud4Tenant -Id <GUID des Tenants>

 

Mit folgendem Kommando kann eine Liste der verfügbaren Tenants ausgelesen werden. Funktioniert nur wenn bereits ein Tenant gesetzt wurde.

Get-Cloud4Tenant

 

Mit folgendem Kommando kann ein bestehender Tenant gelöscht werden. Dies Funktioniert nur wenn alle Ressourcen aus diesem Tenant vorgängig gelöscht wurden.

Remove-Cloud4Tenant -Id <GUID des Tenants>
